GPU 数据结构

场景数据持久存储在 GPU 缓冲区中，CPU 只通过 Upload Buffer 做增量更新：

GPU Data Buffers

SceneBuffer物体世界矩阵、包围盒、LOD 范围
MeshBuffer顶点/索引数据，每个 LOD 级别
MaterialBuffer材质参数、纹理句柄
VisibleBufferCulling 输出：可见性标记
IndirectBuffer压紧后的 DrawIndexedInstancedIndirect 参数
LODBuffer每个物体的当前 LOD 级别

HLSL 对应结构 (GPU 侧)

struct SceneObject {
    float4x4 worldMatrix;
    float3   boundingSphereCenter;
    float    boundingSphereRadius;
    uint     meshIndex;
    uint     materialIndex;
    float    lodDistance[4]; // LOD 0-3 切换距离
};

struct IndirectCommand {
    uint indexCountPerInstance;
    uint instanceCount;
    uint startIndexLocation;
    int  baseVertexLocation;
    uint startInstanceLocation;
};

GPU Culling 管线

// Compute Shader 伪代码
RWStructuredBuffer     VisibleBuffer    : UAV 0;
StructuredBuffer<SceneObject> SceneObjects    : SRV 0;
ConstantBuffer<FrustumData>  Frustum          : CB 0;

[numthreads(64, 1, 1)]
void CullingCS(uint3 id : SV_DispatchThreadID) {
    SceneObject obj = SceneObjects[id.x];

    // 1. 视锥体裁剪
    bool visible = IsInsideFrustum(Frustum, obj.boundingSphereCenter,
                                    obj.boundingSphereRadius);

    // 2. 遮挡裁剪 (可选, 通过 Hi-Z Map)
    if (visible)
        visible = IsNotOccluded(obj.boundingSphereCenter, obj.boundingSphereRadius);

    // 3. Dynamic LOD 选择
    uint lod = SelectLOD(obj, Frustum.CameraPos);

    // 写入可见性 + LOD
    VisibleBuffer[id.x] = visible ? (1 | (lod << 1)) : 0;
}

Indirect Rendering — 可见性压紧 (Compact)

Culling 输出后，需要将可见物体的绘制参数压紧到 IndirectBuffer：

// 使用 AppendBuffer / Counter 做可见性压紧
RWStructuredBuffer<IndirectCommand> IndirectArgs : UAV 0;
AppendStructuredBuffer<uint>        VisibleList  : UAV 1; // 可见物体ID列表

[numthreads(64, 1, 1)]
void CompactCS(uint3 id : SV_DispatchThreadID) {
    uint visibility = VisibleBuffer[id.x];
    if (visibility & 0x1) { // 可见
        uint lod = visibility >> 1;
        SceneObject obj = SceneObjects[id.x];

        // 写入可见物体 ID
        VisibleList.Append(id.x);

        // 根据 LOD 写入 Indirect 参数
        MeshInfo mesh = MeshBuffer[obj.meshIndex][lod];
        IndirectArgs[Counter] = mesh.GetIndirectCommand();
    }
}

GPU Data Management

场景数据管理策略：

  • 持久 GPU 缓冲区 — 物体数据在 GPU 上分配一次，不每帧重新上传
  • CPU 增量更新 — 使用 Upload Buffer + Copy Queue 做逐帧增量更新（移动物体变换、LOD 切换）
  • 双缓冲 — 每帧交替写入，避免 GPU 正在读取时 CPU 写入
  • 稀疏绑定 — 材质/Mesh 数据通过 Bindless 描述符访问，Shader 中用 uint index 索引

Dynamic LOD

LOD 选择在 GPU Culling Dispatch 中同时完成：

  • 距离驱动 — 物体到相机的距离 vs 预设的 LOD 切换距离
  • 屏幕大小驱动 — 基于物体投影到屏幕的像素面积
  • LOD 过渡 — HLSL 中做 dither crossfade，避免 LOD 切换突兀
  • 每个物体在 SceneBuffer 中保存 4 组 LOD Mesh 索引 + 切换距离阈值
