第一层：整体模块架构

5 个 CMake 库 + 严格单向依赖 + C++20 Modules

依赖层次（箭头 = "依赖于"）
                    ┌──────────────────────┐
                    │    Aether.Shaders    │  ← HLSL编译、反射、Shader库
                    │  (ShaderCompiler)    │
                    └────────┬─────────────┘
                             │ depends on
                    ┌────────▼─────────────┐
                    │   Aether.Renderer    │  ← GPU Driven Pipeline
                    │  (GPUDrivenPipeline) │      Culling, Indirect, LOD
                    └────────┬─────────────┘
                    ┌────────▼─────────────┐
                    │  Aether.Resources    │  ← 异步资源加载 (Coroutine)
                    │  (AsyncLoader)       │      Mesh/Texture/Material
                    └────────┬─────────────┘
                    ┌────────▼─────────────┐
                    │     Aether.RHI       │  ← CommandList抽象
                    │  (RHI::Device)       │      D3D12实现 / Vulkan实现
                    │  (RHI::CommandList)  │
                    └────────┬─────────────┘
                    ┌────────▼─────────────┐
                    │     Aether.Core      │  ← 基础类型、数学、内存、Log
                    │  (Math, Memory,      │
                    │   Span, Log)         │
                    └──────────────────────┘

每个库的 C++20 Modules 设计

每个 CMake 库是一个 Module 命名空间，公开的 Module 接口文件（.ixx 或 .cppm）放在 src/<lib>/module/ 下：

CMake Target Module 命名 公开接口 (export)
Aether.Core aether.core Math, Memory, Span, Log, Handle
Aether.RHI aether.rhi Device, CommandList, Buffer, Texture, PipelineState, ShaderBinding
Aether.Resources aether.resources ResourceLoader<T>, CoroutineTask<T>, MeshResource, TextureResource
Aether.Renderer aether.renderer Scene, SceneView, CullingJob, IndirectDraw, LODManager, GPUBufferManager
Aether.Shaders aether.shaders ShaderCompiler, ShaderReflection, ShaderLibrary

关键设计规则

  • 严禁反向依赖 — Renderer 不可直接依赖 Core 以外的下层，下层不可知道上层存在
  • 接口 vs 实现 — RHI 层是纯抽象接口，D3D12/Vulkan 实现在独立源码中通过条件编译选择
  • Modules 命名约定 — 全小写、点分隔，如 aether.rhi.command_list、aether.renderer.scene_view
  • 内部实现不导出 — 不需要跨 Module 访问的类和函数不放在 export 块中
